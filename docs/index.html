Introduction

Cognitive Kernel is an open-source agent system designed to pursue the goal of building generalist autopilots. Unlike copilot systems, which primarily rely on users to provide essential state information (e.g., task descriptions) and assist users by answering questions and auto-completing contents, autopilot systems should complete tasks from start to finish independently, which requires the system to acquire the state information from the environments actively.

To achieve this, an autopilot system must understand user intents, actively gathering necessary information from various real-world sources — such as the open web, local documents, and user-specific long-term memory — and making wise decisions. Cognitive Kernel adopts a model-centric design, where the central policy model (a fine-tuned LLM) initiates interactions with the environment using a combination of atomic actions, such as opening files, clicking buttons, saving intermediate results to memory, or calling the LLM itself. This differs from the widely used environment-centric design, where a task-specific environment with predefined actions is fixed, and the policy model is limited to select an action from a given set of options.

Cognitive Kernel is fully dockerized, ensuring everyone can deploy it privately and securely. We release the system and the backbone model to encourage further research on LLM-driven autopilot systems.

Figure 1: Conceptual comparison of (a) Copilot system, (b) Environment-centric Agent system, and (c) Cognitive Kernel system. After receiving a task, Cognitive Kernel will evaluate whether it has all essential state information to make a sound action. If not, it will actively perceive the missing state information from the environment, which can be a deeper-level self-contained autopilot task.
